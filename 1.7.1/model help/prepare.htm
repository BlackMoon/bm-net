Подготовка данных

На данном этапе создается файл проекта (*.mvf), содержащий основные параметры для визуализации.

Этот файл содержит секции:

  • [in] - входные данные;
  • [out] - секция используется для сохранения настроек;
  • [dim] - геометрические данные объекта;
  • [files] - внешние файлы;
  • [params] - набор параметров объекта (может отсутствовать);
  • [roxar] - гидродинамические параметры объекта (может отсутствовать);
  • [date] - даты для динамических параметров (может отсутствовать).
    -----------------------------------------------

    Состав *.txt файла, описывающего параметр, имеет вид:
    min значениеmax значение
    узел (0, 0)узел(1,0)узел(2,0)…
    узел (0, 1)узел(1,1)узел(2,1)…
    …………………………………………….

    Количество значение в файле - nx * ny * nz + 2(min и max). nz - общее количество слоев.
    -----------------------------------------------

    Состав *.txt файла, описывающего геометрические данные, имеет вид:
    min значениеmax значение
    Узлы кровли
    узел (0, 0)узел(1, 0)узел(2, 0)…
    узел (0, 1)узел(1, 1)узел(2, 1)…
    …………………………………………….
    Узлы подошвы
    узел (0, 0)узел(1, 0)узел(2, 0)…
    узел (0, 1)узел(1, 1)узел(2, 1)…
    …………………………………………….

    Количество значение в файле - 2 * nx * ny * nz + 2(min и max). nz - общее количество слоев.
    -----------------------------------------------

    Состав *.txt файла, описывающего динамические данные, имеет вид:
    MIN значениеMAX значение
    дата 1
    min значениеmax значение
    узел (0, 0)узел(1, 0)узел(2, 0)…
    узел (0, 1)узел(1, 1)узел(2, 1)…
    …………………………………………….
    дата 2
    min значениеmax значение
    узел (0, 0)узел(1, 0)узел(2, 0)…
    узел (0, 1)узел(1, 1)узел(2, 1)…
    …………………………………………….
    дата 3
    min значениеmax значение
    узел (0, 0)узел(1, 0)узел(2, 0)…
    узел (0, 1)узел(1, 1)узел(2, 1)…
    …………………………………………….

    Количество значение в файле - nd *( nx * ny * nz + 2) + 2(MIN и MAX). nd - количество периодов, nz - общее количество слоев
    Прим. MIN, MAX - глобальные ограничения.
    -----------------------------------------------

    Состав *.txt файла, описывающего скважины, имеет вид:
    имя скважины
    х-координатау-координатаабс. отметка глубины
    х-координатау-координатаабс. отметка глубины
    …………………………………………….
    /

    / - обязательный символ
    Прим. После имени скважины может стоять символ *, указывающий на нагнетательную скважину.
